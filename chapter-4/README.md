# 第四章：实战工作流

前三章打完了基础，这一章开始真正干活。

日常开发里 80% 的工作，都可以归结为这五类任务：读懂别人的代码、修 bug、重构、写测试、规划复杂改动。每一类都有它的最佳用法——不是随便问一句就完事，而是有套路的。

本章每一节都有完整的操作流程和可以直接复制的 prompt，跟着做就行。
